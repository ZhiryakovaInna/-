## Описание лабораторной работы

В данной лабораторной работе изучаются основы многопоточного программирования в Java. Рассматриваются три подхода: использование `ExecutorService` и `Callable` для параллельного суммирования массива, создание собственных потоков через наследование от `Thread` с синхронизацией через `join()`, а также применение `CompletableFuture` для асинхронной обработки данных.

---

## Задание 1: Многопоточное суммирование элементов массива

### Описание

Программа вычисляет сумму элементов большого массива (10000 элементов) путём распределения нагрузки между четырьмя потоками.

### Подробное объяснение кода

Создаётся пул из четырёх потоков через `Executors.newFixedThreadPool(4)`. Массив разбивается на четыре блока. Для каждого блока создаётся задача `Callable<Long>`, которая вычисляет сумму на своём участке. Задачи отправляются в пул через `submit()`, возвращаются объекты `Future<Long>`. Основной поток вызывает `future.get()` для получения каждого результата (метод блокируется до завершения задачи). Частичные суммы складываются. В блоке `finally` вызывается `executor.shutdown()` для освобождения ресурсов.

## Задание 2: Поиск максимума в матрице в многопоточном режиме

### Описание

Программа находит наибольший элемент в матрице, выделяя отдельный поток для поиска максимума в каждой строке.

### Подробное объяснение кода

Создаётся класс `RowMaxWorker`, наследуемый от `Thread`. В конструктор передаётся строка матрицы. В методе `run()` перебираются все элементы строки и находится максимум. В `main` для каждой строки создаётся и запускается поток через `start()`. Главный поток вызывает `join()` для каждого рабочего потока — это блокирует выполнение до завершения всех потоков. После этого из каждого рабочего потока через геттер извлекается локальный максимум, и выбирается абсолютный максимум.

## Задание 3: Асинхронная передача партий товаров

### Описание

Программа демонстрирует асинхронную обработку данных: товары накапливаются в партии до веса 150 единиц, затем отправляются на доставку в отдельном потоке.

### Подробное объяснение кода

Цикл перебирает товары, добавляя их в текущую партию и суммируя вес. Когда вес достигает 150, создаётся копия партии и запускается асинхронная задача через `CompletableFuture.runAsync()`. Задача вызывает `deliverBatch()`, который имитирует доставку через `Thread.sleep(1500)`. Каждая задача сохраняется в список. В конце `CompletableFuture.allOf()` объединяет все задачи, а `get()` блокирует главный поток до завершения всех доставок.
