## Описание лабораторной работы

В данной лабораторной работе изучаются коллекции в Java. Рассматриваются основные интерфейсы и реализации (`HashMap`, `ConcurrentHashMap`, `ArrayList`), обобщения (generics) для создания универсальных структур данных, а также применение коллекций для решения прикладных задач: подсчёт частоты слов, реализация стека и система учёта продаж.

---

## Задание 1: Подсчёт частоты слов (TopWords)

### Описание

Программа читает текстовый файл, подсчитывает частоту встречаемости каждого слова и выводит топ-10 самых популярных слов.

### Подробное объяснение кода

Файл считывается построчно через `Scanner`. Каждое слово приводится к нижнему регистру (`toLowerCase()`), затем из него удаляются все знаки препинания с помощью регулярного выражения `[^a-zA-Zа-яА-Я0-9]` — остаются только буквы (латиница и кириллица) и цифры. Для хранения пар «слово — количество» используется `HashMap<String, Integer>`. При добавлении слова применяется метод `getOrDefault(word, 0) + 1`: если слова ещё нет в карте, возвращается 0, затем прибавляется 1 и значение сохраняется. После обработки всего файла набор записей карты (`entrySet()`) переносится в `ArrayList` для сортировки. `Collections.sort()` с компаратором сортирует записи по убыванию количества (`o2.getValue().compareTo(o1.getValue())`). Затем выводится топ-10.

## Задание 2: Реализация обобщённого стека (Stack)

### Описание

Реализована структура данных «Стек» с использованием дженериков, поддерживающая операции добавления (`push`), удаления (`pop`) и просмотра верхнего элемента (`peek`).

### Подробное объяснение кода

Класс `Stack<T>` параметризован типом `T`. Внутри используется массив `data` типа `T[]`. В Java нельзя напрямую создать массив обобщённого типа, поэтому создаётся массив `Object` с нужной ёмкостью, который приводится к `T[]` (аннотация `@SuppressWarnings("unchecked")` подавляет предупреждение компилятора). Поле `size` хранит текущее количество элементов. Метод `push()` проверяет, не достигнута ли максимальная ёмкость, затем помещает элемент на вершину стека (по индексу `size`) и увеличивает `size`. Метод `pop()` проверяет, не пуст ли стек, затем уменьшает `size`, сохраняет элемент по новому индексу `size`, обнуляет ячейку (чтобы избежать утечек памяти) и возвращает элемент. Метод `peek()` возвращает элемент по индексу `size - 1` без удаления.

## Задание 3: Система учёта продаж (SalesTracker)

### Описание

Создана система для учёта проданных товаров, расчёта общей выручки и поиска наиболее популярного товара.

### Подробное объяснение кода

Для хранения количества продаж используется `ConcurrentHashMap<String, Integer>` — она обеспечивает потокобезопасность (важно для систем учёта, где операции могут выполняться одновременно). Отдельная карта `productPrices` (обычный `HashMap`) хранит цены товаров. Метод `recordSale()` использует метод `merge()`: он принимает ключ (название товара), новое значение (количество) и функцию для объединения (в данном случае `Integer::sum` — складывает текущее значение с новым). Это атомарная операция, безопасная в многопоточной среде. Метод `calculateTotalSales()` перебирает записи в `salesCount`, для каждого товара получает цену из `productPrices` (если цены нет — берётся 0.0) и умножает на количество, суммируя результат. Метод `getMostPopularProduct()` линейно обходит карту продаж, отслеживая максимальное значение и соответствующий ключ.
