## Описание лабораторной работы

В данной лабораторной работе разработана система автоматизированной многопоточной обработки данных, использующая рефлексию (Reflection API) и аннотации для динамического обнаружения и выполнения логики обработки.

---

## Задание 1: Создание аннотации DataProcessor

### Описание

Создана пользовательская аннотация `@DataProcessor`, которая используется для маркировки методов, предназначенных для обработки данных.

### Подробное объяснение кода

Аннотация помечена `@Retention(RetentionPolicy.RUNTIME)` — это критически важно, так как указывает Java сохранить информацию об аннотации после компиляции, чтобы программа могла "увидеть" её через рефлексию во время выполнения. `@Target(ElementType.METHOD)` ограничивает применение аннотации только методами.

## Задание 2: Класс DataManager (центральный узел системы)

### Описание

Класс управляет загрузкой данных, регистрацией обработчиков и многопоточной обработкой. Использует рефлексию для обнаружения методов, помеченных аннотацией `@DataProcessor`, и запускает их в отдельных потоках.

### Подробное объяснение кода

Класс содержит три списка: `sourceData` (исходные данные), `processors` (зарегистрированные объекты-обработчики) и `resultData` (результаты обработки, обёрнутый в `Collections.synchronizedList()` для потокобезопасности). Метод `processData()` создаёт пул потоков через `ExecutorService`. Для каждого зарегистрированного обработчика через рефлексию получаются все его методы (`getDeclaredMethods()`). Если метод помечен аннотацией `@DataProcessor`, он запускается в отдельном потоке через `executor.submit()`. Внутри задачи вызывается `method.invoke(processor, sourceData)`, который выполняет метод обработчика, передавая ему исходные данные. Результат добавляется в `resultData`. После запуска всех задач вызывается `executor.shutdown()` и `awaitTermination()` для ожидания завершения.

## Задание 3: Классы обработки (PlanetFilter и PlanetTransformer)

### Описание

Классы содержат конкретную логику обработки данных. Методы помечены аннотацией `@DataProcessor`, что делает их обнаруживаемыми для `DataManager`.

### Подробное объяснение кода

`PlanetFilter.filterLongNames()` с помощью Stream API отбирает названия планет длиннее 6 символов (`filter(s -> s.length() > 6)`). `PlanetTransformer.transformToUpper()` преобразует каждое название в верхний регистр (`map(String::toUpperCase)`) и добавляет эмодзи "🪐". Оба метода возвращают обработанные списки.

## Задание 4: Класс Main

### Описание

Точка входа в программу. Создаёт `DataManager`, регистрирует обработчики, загружает данные и запускает обработку.

### Подробное объяснение кода

Создаётся объект `DataManager`. Через `registerDataProcessor()` регистрируются экземпляры `PlanetFilter` и `PlanetTransformer`. `loadData()` передаёт список названий планет. `processData()` запускает многопоточную обработку. `saveData()` выводит результаты.
