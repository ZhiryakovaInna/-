## Описание лабораторной работы

В данной лабораторной работе изучаются принципы работы хеш-таблиц. Реализуется собственный класс `HashTable` для хранения пар «ключ-значение» с использованием метода цепочек для разрешения коллизий. Также применяется стандартный класс `HashMap` из библиотеки Java для решения прикладной задачи — создания телефонной книги.

---

## Задание 1: Реализация структуры данных HashTable

### Описание

Реализована собственная хеш-таблица с поддержкой обобщённых типов (generics). Таблица хранит пары «ключ-значение», использует метод цепочек (связные списки) для разрешения коллизий и поддерживает операции добавления, получения и удаления элементов.

### Подробное объяснение кода

Внутри класса `HashTable` создаётся вспомогательный класс `Entry`, который хранит пару «ключ-значение». Основное хранилище — это массив связных списков `LinkedList<Entry<K, V>>[]`, где каждая ячейка массива соответствует одной корзине (bucket). Размер массива фиксирован и равен 10.

Хеш-функция `hash()` вычисляет индекс корзины для ключа: берётся `hashCode()` ключа, затем абсолютное значение и остаток от деления на размер массива. Это гарантирует, что индекс всегда будет в пределах от 0 до 9.

Метод `put()` добавляет новую пару или обновляет существующую. Сначала вычисляется индекс, затем если по этому индексу нет списка, он создаётся. Далее выполняется поиск записи с таким же ключом: если найдена — обновляется значение, если нет — в конец списка добавляется новая запись, и счётчик размера увеличивается.

Метод `get()` ищет значение по ключу. Вычисляется индекс, затем выполняется перебор списка в этой корзине. Если запись с нужным ключом найдена, возвращается её значение. Если нет — возвращается `null`.

Метод `remove()` удаляет запись по ключу. Находит нужную корзину и удаляет из списка запись с совпадающим ключом, после чего уменьшает счётчик размера.

## Задание 2: Работа с коллекциями (Телефонная книга)

### Описание

С использованием стандартного класса `HashMap` реализована телефонная книга. Ключом является номер телефона (строка), значением — объект класса `Contact`, содержащий имя, фамилию и email. Демонстрируются операции добавления, поиска, удаления и вывода всех контактов.

### Подробное объяснение кода

Класс `Contact` представляет контактную информацию: имя, фамилия, email. В нём переопределён метод `toString()` для удобного вывода в формате: «Имя Фамилия (Email: адрес)».

В методе `main` создаётся объект `HashMap<String, Contact>`, где ключ — номер телефона (строка), значение — объект `Contact`. Метод `put()` добавляет несколько контактов в телефонную книгу. Метод `size()` показывает количество записей.

Для поиска контакта используется `containsKey()` для проверки наличия ключа и `get()` для получения значения по ключу. Удаление выполняется методом `remove()`. Для вывода всех контактов используется `keySet()`, который возвращает все ключи, после чего в цикле для каждого ключа получается соответствующий контакт и выводится на экран.
