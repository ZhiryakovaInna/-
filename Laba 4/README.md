## Описание лабораторной работы

В данной лабораторной работе изучаются механизмы обработки исключений в Java. Рассматриваются перехват стандартных исключений при работе с массивами и числами, безопасная работа с файлами через try-with-resources, а также создание собственных классов исключений и логирование ошибок в файл.

---

## Задание 1: Обработка исключений при работе с массивами

### Описание

Программа демонстрирует обработку `NumberFormatException` (преобразование строки в число) и `ArrayIndexOutOfBoundsException` (выход за границы массива). Для корректных чисел вычисляется сумма и среднее арифметическое.

### Подробное объяснение кода

В цикле `for` перебирается массив строк. В блоке `try` каждая строка преобразуется в число через `Integer.parseInt()`. Если преобразование успешно, число добавляется к сумме. Если возникает `NumberFormatException`, исключение перехватывается, и выводится сообщение об ошибке — программа продолжает работу.

Отдельным блоком `try-catch` выполняется попытка обращения к индексу за пределами массива (например, `arr[arr.length]`). Возникает `ArrayIndexOutOfBoundsException`, которое перехватывается и выводится сообщение. В конце выводится сумма, количество и среднее арифметическое корректных чисел.

## Задание 2: Работа с файлами

### Описание

Программа копирует содержимое одного файла в другой с использованием конструкции `try-with-resources`, автоматически закрывающей потоки ввода-вывода.

### Подробное объяснение кода

В круглых скобках после `try` создаются объекты `FileInputStream` (для чтения) и `FileOutputStream` (для записи). Эти потоки будут закрыты автоматически при выходе из блока.

Внутри блока `try` организован цикл: метод `in.read()` считывает байт за байтом из исходного файла до тех пор, пока не вернёт `-1` (конец файла). Каждый байт сразу записывается в выходной файл через `out.write()`. При возникновении `IOException` (например, файл не найден) исключение перехватывается в `catch`, выводится сообщение об ошибке.

## Задание 3: Собственное исключение и логирование

### Описание

Создаётся собственный класс исключения `Task_3_1`. Калькулятор выбрасывает это исключение при делении на ноль или при неподдерживаемой операции. Информация об ошибке записывается в лог-файл.

### Подробное объяснение кода

Класс `Task_3_1` наследуется от `Exception`. Конструктор принимает сообщение и передаёт его родительскому классу.

Метод `calculate()` принимает два числа и операцию. При делении проверяется, равен ли второй аргумент нулю — если да, выбрасывается `Task_3_1`. Если операция не поддерживается — также выбрасывается исключение.

В методе `main` вызовы `calculate()` обёрнуты в `try-catch`. При перехвате исключения вызывается метод `logErrorToFile()`, который через `PrintWriter` записывает в файл `errors.log` тип ошибки и сообщение. Для записи также используется `try-with-resources`, файл открывается в режиме добавления.
