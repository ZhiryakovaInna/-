## Описание лабораторной работы

В данной лабораторной работе реализуется иерархия классов на примере предметной области «Мебель». Целью работы является закрепление принципов объектно-ориентированного программирования (ООП): абстракции, инкапсуляции, наследования и полиморфизма.

---

## Задание 1: Класс Furniture (Базовый абстрактный класс)

### Описание

Базовый абстрактный класс `Furniture` служит шаблоном для всех конкретных видов мебели. Содержит общие поля (материал, цвет, цена), статический счётчик объектов и абстрактный метод `use()`.

### Подробное объяснение кода

Класс объявлен как `abstract`, поэтому нельзя напрямую создать его экземпляр — он нужен только как основа для наследников. Поля `material`, `color`, `price` имеют модификатор `private` (инкапсуляция), доступ к ним осуществляется через геттеры и сеттеры. Статическая переменная `count` принадлежит не конкретному объекту, а всему классу, и увеличивается в конструкторе при создании каждого нового объекта. Это позволяет отслеживать общее количество созданной мебели (столов, стульев, кроватей вместе). Абстрактный метод `use()` не имеет реализации в базовом классе — каждый неабстрактный наследник обязан предоставить свою реализацию. Метод `describe()` выводит информацию о мебели.

## Задание 2: Класс Table (Наследник)

### Описание

Класс `Table` расширяет `Furniture`, добавляя свойство `legsCount` (количество ножек). Демонстрирует наследование, переопределение и перегрузку методов.

### Подробное объяснение кода

Класс использует `extends Furniture` для наследования всех полей и методов родителя. Конструктор вызывает `super(material, color, price)`, который инициализирует унаследованные поля, затем устанавливает собственное поле `legsCount`. Аннотация `@Override` указывает, что метод `use()` переопределён: теперь при вызове у объекта `Table` выводится сообщение «За столом можно работать или обедать». Также в классе присутствует перегруженная версия `use(String activity)` — метод с тем же именем, но другим набором параметров, что позволяет передавать дополнительную информацию (например, для чего используется стол).

## Задание 3: Класс Chair (Наследник)

### Описание

Класс `Chair` расширяет `Furniture`, добавляя свойство `hasBackrest` (наличие спинки).

### Подробное объяснение кода

`Chair` наследует от `Furniture`. Конструктор через `super()` передаёт материал, цвет и цену родителю, а затем инициализирует собственное поле `hasBackrest`. Метод `use()` переопределён: при вызове выводится сообщение «На стуле можно сидеть». Благодаря наследованию объект `Chair` автоматически получает метод `describe()` и статический счётчик `count` из родительского класса.

## Задание 4: Класс Bed (Наследник)

### Описание

Класс `Bed` расширяет `Furniture`, добавляя свойство `size` (размер кровати).

### Подробное объяснение кода

Аналогично предыдущим классам, `Bed` наследует от `Furniture`. Конструктор вызывает `super()` для инициализации общих полей и устанавливает `size`. Метод `use()` переопределён: выводится сообщение «На кровати можно спать». Это демонстрирует полиморфизм — один и тот же метод `use()` ведёт себя по-разному в зависимости от того, у объекта какого класса он вызывается (стол, стул или кровать).

## Задание 5: Класс Main

### Описание

Главный класс, демонстрирующий работу иерархии: создание объектов, вызов методов и работу статического счётчика.

### Подробное объяснение кода

В методе `main` создаются три объекта: `Table`, `Chair`, `Bed`. При каждом создании конструктор наследника вызывает `super()`, который увеличивает статическую переменную `count` в классе `Furniture`. Вызов `table.describe()` демонстрирует наследование — метод `describe()` не определён в `Table`, но доступен от родителя. Вызовы `table.use()` и `chair.use()` показывают полиморфизм — каждый объект выполняет свою версию метода. Вызов `table.use("Черчение проекта")` — это перегруженный метод, специфичный только для класса `Table`. `Furniture.getCount()` выводит общее количество созданных объектов мебели (3), что подтверждает, что статическая переменная общая для всей иерархии.
